## Secretary of the Interior's Standards — Always Cite by Number
1. Compatible use
2. Historic character retained
3. No false sense of history
4. Changes with acquired significance retained
5. Distinctive materials/features/finishes preserved ← most cited
6. Deteriorated features repaired not replaced ← most cited
7. No damaging chemical/physical treatments
8. Archaeological resources protected
9. New additions distinguishable and not destroying historic material ← most cited
10. New additions reversible

## Narrative Structure (always in this order)
1. Project Description — property, address, construction date, historic designation
2. Statement of Significance — why listed, period of significance
3. Existing Conditions — current state of historic fabric being affected
4. Proposed Scope of Work — what, materials, methods
5. Standards Compliance Analysis — cite each Standard by number
6. Reversibility Statement
7. Material Specifications — match-existing protocol

## Language Rules
- Passive, formal, regulatory tone throughout
- Never: "we will" → use: "the proposed work will"
- Never: "old material" → use: "historic fabric"
- Never: "original features" → use: "character-defining features"
- Never: "matching" for new additions → use: "compatible"
- Never: "restore" unless meeting Standards definition of restoration
- Never: "modern" as a positive descriptor

## Code & Accessibility Cross-Check
Historic preservation projects still must meet current code. Before drafting any
narrative involving the following, check `hot.md` in this folder:
- ADA accessibility (mezzanines, thresholds, ramps in historic spaces)
- Egress from historic spaces (stair enclosures, exit widths)
- Fire ratings for new walls/floors introduced into historic fabric
- Sprinkler system introduction (wet vs. dry, concealed head options)
- Change of occupancy (triggers IBC §1011 + CBC Chapter 13-196)

**ADA mezzanine exception:** Under IBC §1104.4, mezzanines ≤3,000 SF in existing
buildings may be exempt from accessible route requirement. This is frequently
applicable on historic preservation projects. Cite this section explicitly in
narratives when it applies rather than specifying a costly lift.

**Key rule:** Never recommend a code solution in a SHPO narrative without confirming
the code path is correct. An incorrect code cite can trigger SHPO comments and delay.
