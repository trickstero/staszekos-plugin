# Code Research — Session Index

## How to use this project
At the start of every session:
1. Read `_memory.md` — standing instructions, research sequence, case log, patterns
2. Read all files in `_inputs/` — project brief, jurisdiction, code documents
3. Perform research per standing instructions in `_memory.md`
4. Save output to `_outputs/` using the naming convention
5. Append one line to the Case Log in `_memory.md`
6. Append any new AHJ interpretations or code quirks to Learned Patterns in `_memory.md`

## Folder structure
- `_inputs/` — project briefs, code excerpts, AHJ correspondence
- `_outputs/` — code matrices, compliance checklists, AHJ notes
- `_memory.md` — running project memory, instructions, case log, patterns
